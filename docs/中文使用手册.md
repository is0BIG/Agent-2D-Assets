# Agent-2D-Assets 中文使用手册

Agent-2D-Assets 是一套面向 Codex 的 2D 游戏资产工作流。它不是单纯的提示词合集，而是让 Codex 负责资产规划与图像生成，再用本地 Python 脚本完成去背、切帧、对齐、质量检查和导出。

## 适合做什么

- 角色、怪物、NPC、召唤物等 2D sprite。
- idle、walk、run、attack、cast、projectile、impact 等动画帧。
- 透明 PNG、透明 sprite sheet、GIF 预览。
- 地图 base、分层地图、prop pack、单独 props、地图预览。
- Godot、Unity、Phaser、Tiled、LDtk 等项目需要的素材和元数据草案。

## 安装

在 Windows PowerShell 中：

```powershell
git clone https://github.com/is0BIG/Agent-2D-Assets.git
cd .\Agent-2D-Assets
python -m pip install -r .\requirements.txt
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.codex\skills" | Out-Null
Copy-Item -Recurse -Force ".\skills\*" "$env:USERPROFILE\.codex\skills\"
```

在 macOS 或 Linux 中：

```bash
git clone https://github.com/is0BIG/Agent-2D-Assets.git
cd ./Agent-2D-Assets
python3 -m pip install -r ./requirements.txt
mkdir -p ~/.codex/skills
cp -R ./skills/* ~/.codex/skills/
```

安装后请新开一个 Codex session，让 `$generate2dsprite` 和 `$generate2dmap` 被重新加载。

## 使用 `$generate2dsprite`

`$generate2dsprite` 用于生成和处理独立 sprite 资产。

推荐提示词结构：

```text
Use $generate2dsprite to create [对象] with [动作], [视角], [风格], [输出形式].
```

常用参数含义：

- 对象：player、npc、creature、character、spell、projectile、impact、prop、fx。
- 动作：single、idle、walk、run、attack、cast、projectile、impact、death。
- 视角：top-down、side-view、3/4。
- sheet：2x2、2x3、2x4、3x3、4x4。
- 输出：transparent PNG、frame PNGs、GIF preview、engine atlas。

示例：

```text
Use $generate2dsprite to create a side-view fire mage spell bundle with cast animation, projectile, and impact FX, pixel-inspired style, transparent PNG and GIF exports.
```

```text
Use $generate2dsprite to create a 4-direction top-down walk sheet for a tiny armored knight, 4x4 grid, consistent scale, transparent output.
```

```text
Use $generate2dsprite to create a 2x2 impact animation for a blue ice burst, transparent output and GIF preview.
```

### Sprite 工作流

1. Codex 判断资产类型、动作、sheet 形状和风格。
2. Codex 使用内置图像生成创建纯洋红背景的 raw sheet。
3. 本地脚本把洋红背景变成透明背景。
4. 脚本按网格切帧，统一缩放和对齐。
5. 脚本导出透明 sheet、单帧 PNG、GIF 和 `pipeline-meta.json`。
6. Codex 根据元数据检查是否贴边、是否切帧失败、是否需要重新生成。

### Sprite 输出结构

一次普通 2x2 动画通常会输出：

```text
raw-sheet.png
raw-sheet-clean.png
sheet-transparent.png
animation.gif
frame-1.png
frame-2.png
frame-3.png
frame-4.png
prompt-used.txt
pipeline-meta.json
```

`pipeline-meta.json` 中重要字段：

- `rows` / `cols`：网格行列数。
- `cell_size`：导出单帧尺寸。
- `frame_labels`：每帧文件名。
- `frames`：每帧 bbox、组件数量、输出尺寸和贴边状态。
- `edge_touch_frames`：贴边帧列表。正常情况下应为空。
- `prompt`：本次实际使用的提示词。

## 使用 `$generate2dmap`

`$generate2dmap` 用于地图、场景和关卡资产。

推荐提示词结构：

```text
Use $generate2dmap to create [地图类型] for [玩法/引擎], with [分层/道具/碰撞/出口/预览].
```

常见地图模式：

- `tile_mode`：适合 RPG、宝可梦式地图、战术格子、可编辑 tilemap。
- `scene_mode`：适合塔防、幸存者 arena、俯视角探索场景。
- `side_scroll_mode`：适合横版平台、跑酷、横版动作、类银河城。
- `grid_mode`：适合战棋、工厂、棋盘、卡牌战场。
- `room_chunk_mode`：适合 Roguelike 房间块、程序化拼接地图。
- `baked_scene_mode`：适合单张背景图、战斗背景、菜单、视觉小说场景。

示例：

```text
Use $generate2dmap to create a top-down RPG village entrance map with a ground-only base, separated trees and signs, collision blockers, player spawn, exit zones, and a layered preview.
```

```text
Use $generate2dmap to create a side-scroll cyberpunk canal stage with parallax layers, separate platform objects, hazards, camera bounds, collision metadata, and a stage preview.
```

```text
Use $generate2dmap to create a tower-defense forest pass map with route metadata, build slots, separated props, blockers, spawn and exit hooks, and a flattened preview.
```

### 地图工作流

1. 先确定地图模式和引擎目标。
2. 对可玩地图，先生成 foundation-only base，不把可交互道具、门、敌人、前景遮挡物烘焙进去。
3. 生成 dressed reference 或 stage reference，用作布局参考。
4. 生成或切割单独 props、平台、障碍、门、出口、hazard。
5. 编写 placement、collision、zones、spawns、camera bounds 等结构化数据。
6. 合成 QA preview，检查尺寸、透明通道、道具贴边、碰撞区和出口。

### 地图输出结构

分层地图通常包含：

```text
base.png
dressed-reference.png
props/
prop-pack.json
placements.json
collision.json
zones.json
layered-preview.png
```

横版地图通常还会包含：

```text
sky.png
far-bg.png
mid-bg.png
near-bg.png
foreground-overlay.png
objects.json
stage-preview.png
```

## 本地脚本用法

通常你不需要手动运行脚本，Codex 会替你调用。但调试时可以直接使用。

### 导入最新 image_gen 输出

内置图像生成通常会把图片放在 `$CODEX_HOME/generated_images`，或默认的 `~/.codex/generated_images`。可以用导入脚本把最新 PNG 复制到当前项目输出目录：

```powershell
python .\skills\generate2dsprite\scripts\import_generated_image.py `
  --output .\outputs\my-run\raw-sheet.png
```

如果最新图片不是你要的，可以使用 `--latest-index 1` 选择第二新的图片，或用 `--session` 指定某个生成目录。

也可以使用 provider 抽象入口，从本地文件或 Codex 生成目录解析 raw 图：

```powershell
python .\skills\generate2dsprite\scripts\image_provider.py `
  --provider local_file `
  --source .\somewhere\raw.png `
  --output .\outputs\my-run\raw-sheet.png
```

### 处理 sprite sheet

```powershell
python .\skills\generate2dsprite\scripts\generate2dsprite.py process `
  --input .\raw-sheet.png `
  --target creature `
  --mode idle `
  --output-dir .\out\slime-idle `
  --rows 2 `
  --cols 2 `
  --shared-scale `
  --align bottom `
  --component-mode all `
  --edge-feather 80 `
  --despill `
  --apng `
  --webp `
  --reject-edge-touch
```

常用选项：

- `--rows` / `--cols`：指定网格。
- `--cell-size`：导出单帧尺寸。
- `--fit-scale`：主体在单帧里的缩放比例。
- `--align center|bottom|feet`：对齐方式。
- `--shared-scale`：所有帧使用同一缩放比例。
- `--component-mode all|largest`：保留全部组件或只保留最大主体。
- `--key-color`：指定背景 key 色，默认 `#ff00ff`。
- `--tolerance`：洋红去背容差；是 `--threshold` 的更直观别名。
- `--edge-feather`：对抗锯齿边缘做半透明软化。
- `--despill`：把边缘洋红污染反混色为接近主体的颜色。
- `--apng`：额外导出 APNG 动画预览。
- `--webp`：额外导出无损 animated WebP 预览。
- `--reject-edge-touch`：检测到贴边时直接失败。
- `--prompt-file`：把实际提示词写入产物和元数据。

### 生成 QC HTML 报告

```powershell
python .\skills\generate2dsprite\scripts\generate_qc_report.py `
  --input-dir .\outputs\my-run\processed
```

输出 `qc-report.html`，可以在浏览器中查看帧预览、bbox、输出尺寸、贴边状态和完整 metadata。

### 切割 prop pack

```powershell
python .\skills\generate2dmap\scripts\extract_prop_pack.py `
  --input .\prop-pack.png `
  --rows 3 `
  --cols 3 `
  --output-dir .\out\props `
  --labels rock,barrel,lamp,sign,crate,bush,stump,pot,flower `
  --edge-feather 80 `
  --despill `
  --reject-edge-touch
```

输出会包含每个 prop 的独立透明 PNG，以及 `prop-pack.json` manifest。

### 合成分层预览

```powershell
python .\skills\generate2dmap\scripts\compose_layered_preview.py `
  --base .\base.png `
  --placements .\placements.json `
  --output .\layered-preview.png `
  --report .\preview-report.json
```

`placements.json` 可以是列表，也可以是包含 `props`、`objects` 或 `foreground` 的对象。

## 推荐实践

- 角色动画优先用 2x2、2x3、2x4 等多行网格，不要直接让图像模型生成 1x4 长条角色动画。
- 主角或高价值角色的 idle、run、attack、cast 分开生成，检查通过后再组 atlas。
- 攻击主体和大型 slash、projectile、impact 最好分开生成，运行时叠加。
- 地图 base 只放地面、路、水、低矮地形，不要把可交互对象烘焙进 base。
- prop pack 只放小型紧凑物件。桥、平台、门、墙、长条 hazard、建筑等应单独生成或用 strip/tile 表达。
- 每次保留 `prompt-used.txt` 和 `pipeline-meta.json`，方便复盘和重新处理。

## 常见问题

### GIF 看起来是洋红背景，是否没有透明？

GIF 透明依赖调色板索引。有些预览器会用洋红显示透明色。请优先检查 `sheet-transparent.png` 或单帧 PNG。如果 PNG 是透明的，通常素材本身可用。

### 透明边缘有紫色残留怎么办？

这是洋红背景和主体边缘抗锯齿混合造成的。重新处理 raw sheet 时增加软边和去污染参数：

```powershell
python .\skills\generate2dsprite\scripts\generate2dsprite.py process `
  --input .\raw-sheet.png `
  --target creature `
  --mode idle `
  --output-dir .\out\cleaned `
  --rows 2 `
  --cols 2 `
  --tolerance 80 `
  --edge-threshold 80 `
  --edge-feather 80 `
  --despill
```

如果主体本身含有紫色或粉色，请降低 `--tolerance` 和 `--edge-feather`，避免误删主体颜色。

### 为什么脚本提示图片尺寸不能被网格整除？

处理器需要每个 cell 完全等宽等高。比如 2x2 网格要求图片宽高都能被 2 整除。请重新导出、裁剪、pad 或 resize raw sheet。

### 什么时候用 `component-mode=largest`？

用于角色主体、hero body、单个道具等只应保留一个主体的场景。它会丢弃小碎片和噪点。

### 什么时候用 `component-mode=all`？

用于 projectile、impact、aura、spell FX，或故意有多个分离组件的效果。

### `edge_touch_frames` 不为空怎么办？

说明至少有一帧贴到 cell 边缘。优先重新生成 raw sheet，并在提示词里要求主体只占 cell 中央 60% 到 70%，四周保留洋红边距。也可以降低 `fit-scale`，但如果 raw 图本身已经被裁切，最好重新生成。

## 开发与验证

运行轻量测试：

```powershell
python -m unittest discover -s tests
```

测试覆盖：

- `--prompt-file` 是否写入 `pipeline-meta.json`。

- `component-mode=largest` 是否真正丢弃小噪点。
