# Game Audio Contract

Generated or processed audio should include:

- WAV file, 44.1 kHz, mono, 16-bit PCM by default.
- Engine-safe lowercase filename.
- JSON metadata with id, path, duration, sample rate, channels, peak, RMS, clipping count, and loop flag.
- For packs, an `audio-pack.json` manifest containing every sound entry.

Recommended levels:

- Avoid clipping: `clipping_samples` should be `0`.
- Normalize most SFX to around `-1 dBFS`.
- Keep UI sounds short, usually `0.05s` to `0.2s`.
- Keep gameplay action sounds under `1s` unless intentionally long.

Engine notes:

- Godot can import WAV directly as `AudioStreamWAV`.
- Unity can import WAV directly as `AudioClip`.
- For web games, WAV is useful during development; compressed formats can be generated by downstream tooling.
